# DoENGINE Philosophy

DoENGINE is built on the principles of Doing, Motion, and Vibration. Every system, process, and interaction is modeled as a frequency, a movement, or an act of creation. The nomenclature is unique:
- Doer: Any active agent (player, AI, system)
- Motif: A pattern of motion or behavior
- Vibe: The frequency or energy state of a system
- Trace: A permanent, encrypted record in hardware
- Resonance: Synchronization of systems or agents
- Pulse: A discrete tick or event in the engine
- Flow: Continuous change or transformation

This philosophy guides all code, metadata, and hardware traces.
