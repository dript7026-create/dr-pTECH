# IllusionCanvasInteractive

IllusionCanvasInteractive is a standalone 2.5D pseudo-3D, 2D pixel-art hybrid runtime for drIpTECH projects.

It combines three existing directions in this workspace without collapsing them into one codebase:

- ORBEngine-inspired projection rules for layered pseudo-3D presentation and recursive-feeling spaces.
- DoENGINE-style runtime manifest bucketing so the engine can ingest the same production asset stream used elsewhere in the workspace.
- EgoSphere-backed encounter interpretation, with a dedicated `godAI` conductor that shapes pressure, mercy windows, and world pacing.

The native project format is `.iig`, short for `illusioninteractivegame`.

## What Is Included

- A standalone Tkinter-based runtime app.
- A standalone Tkinter-based `.iig` studio editor.
- A validated `.iig` schema loader and saver.
- A lightweight combat and progression simulation for `aridfeihth`.
- An IllusionCanvas runtime manifest builder for the clip-blend-id-do pipeline.
- A sample vertical-slice `.iig` file.
- A native EgoSphere bridge path with auto-build support for `egosphere.dll`.
- A gothic UI skin pipeline that can consume generated Recraft GUI shells and studio session metadata.

## Running The Standalone App

From Windows Explorer or a terminal, run:

```bat
IllusionCanvasInteractive\IllusionCanvasInteractive.cmd
```

Or run the Python entry point directly:

```powershell
py -3 IllusionCanvasInteractive\run_illusioncanvas.py IllusionCanvasInteractive\sample_games\aridfeihth_vertical_slice.iig
```

## Running The Studio

```bat
IllusionCanvasInteractive\IllusionCanvasStudio.cmd
```

Or:

```powershell
py -3 IllusionCanvasInteractive\run_illusioncanvas_studio.py IllusionCanvasInteractive\sample_games\aridfeihth_vertical_slice.iig
```

## Controls

- `Left` / `Right`: move
- `Z`: primary attack
- `X`: Burst pet action
- `C`: toggle Chorus pet
- `Space`: dodge
- `E`: rescue/interact
- `R`: rest at refuge rooms
- `V`: attempt bond weave

## `.iig` Format

`.iig` is JSON with an engine-facing schema.

Required top-level sections:

- `format`
- `metadata`
- `integrations`
- `rendering`
- `gameplay`
- `world`
- `player`
- `pets`

The format is intentionally plain JSON so it stays deterministic and pipeline-friendly.

## Pipeline Integration

The Bango clip-blend-id-do pipeline now emits:

- `generated/clip_blend_id/illusioncanvas_runtime_manifest.json`
- `generated/clip_blend_id/aridfeihth_illusioncanvas_bundle.iig`

Those outputs are generated by:

- `IllusionCanvasInteractive/tools/build_illusioncanvas_bundle.py`

## Native EgoSphere Bridge

The runtime now prefers a real native EgoSphere path. If `egosphere/egosphere.dll` is missing and a C compiler is available on `PATH`, the bridge auto-builds it.

Manual build:

```powershell
py -3 IllusionCanvasInteractive\tools\build_egosphere_bridge.py
```

## GUI Skin Pass

The Recraft-facing GUI asset contract lives at `IllusionCanvasInteractive/recraft/illusioncanvas_gui_1000_credit_manifest.json` and can be regenerated by:

```powershell
py -3 IllusionCanvasInteractive\tools\build_illusioncanvas_gui_manifest.py
```

To build the runtime-consumable UI skin metadata from that manifest:

```powershell
py -3 IllusionCanvasInteractive\tools\build_illusioncanvas_ui_skin.py --manifest IllusionCanvasInteractive\recraft\illusioncanvas_gui_1000_credit_manifest.json --out IllusionCanvasInteractive\generated\ui\illusioncanvas_gothic_ui_skin.json
```

## Design Intent

This runtime is not a copy of ORBEngine or DoENGINE. It is a bridge layer that borrows useful structure from both while staying standalone and project-specific.
