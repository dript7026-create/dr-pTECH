TOMMY GOOMBA — Complete Asset Specifications

Purpose
-------
This document lists *individual* graphical assets required for the Tommy Goomba project. Each entry contains exact export size(s), frame counts, animation timing, palette and shade usage (Game Boy 4-shade indexed), naming conventions, collision/hitbox metadata, anchor/pivot rules, usage context, and artist notes for faithful implementation in the GBDK build pipeline.

General conventions
-------------------
- Color depth: 2 bpp (4 shades) compatible with original Game Boy palette.
- File format: lossless indexed PNG at export. Master source may be layered PSD/PNG.
- Tile grid: 8x8 tiles. Sprite widths/heights must be multiples of 8 (8, 16, 24, 32).
- Naming: category_assetname_variant[_framecount]_[w]x[h].png (e.g., player_idle_4_16x16.png)
- Export units: pixels (WxH), frames, animation frame duration in game ticks (default 1 tick = 1/60s or engine tick rate — specify per entry).
- Hitbox JSON: for each multi-tile sprite, provide a parallel .json with keys: boundingBox {x,y,w,h}, attackBoxes[], hurtBoxes[], pivot {x,y}, anchorTile {col,row}, collisionMask (tile-level booleans).
- Palette indices: 0=white (lightest), 1=light gray, 2=dark gray, 3=black (darkest). Avoid using alpha for visible pixels; use transparency only for padding.

12) Export Recommendations & Pipeline Notes
- Master sources: keep PSD/ layered PNG with separate color/outline/alpha layers; also provide flattened indexed PNGs for engine import.
- Tile packing: supply both individual tile PNGs and a packed tilesheet PNG plus a JSON index map (tileIndex -> x,y in sheet).
- Frame ordering: left-to-right in the PNG file; also provide separate folder per animation with single-frame PNGs if that fits artist workflow.
- Metadata files: for every multi-frame sprite deliver a .meta.json with:
  - name: string
  - frames: int
  - frameDurations: [ints]
  - pivot: {x,y}
  - boundingBox: {x,y,w,h}
  - attackBoxes: [{x,y,w,h}] (if applicable)
  - tags: ["enemy","boss","ui"]
  - paletteHint: "use shade 3 for outlines, shade 2 for body"
