# TOMMY GOOMBA — Complete Asset Specification
Version: 1.0  
Target: Nintendo Game Boy / Game Boy Color-compatible ROM (`.gb`)

---

## 1) Technical Asset Pipeline (Authoring → Build)

## 1.1 Art Source Formats (Editable)
- **Aseprite source:** `.aseprite`
- **Layered backup:** `.psd` (optional)
- **Reference sheets:** `.png` (indexed palette, no alpha gradients)

## 1.2 Runtime Art Formats (Build Outputs)
- **Tile/sprite data:** generated C arrays (`.c/.h`) via converter
- **Recommended converter:** `png2asset` (GBDK pipeline)
- **Sprite size mode:** 8x16 (preferred for characters), 8x8 for UI/effects
- **Palette:** 4-shade DMG-safe values (required baseline), optional CGB palette extension

## 1.3 Map/Level Formats
- **Authoring:** Tiled `.tmx` or LDtk `.ldtk`
- **Export:** CSV/JSON → compile-time C arrays (`stage_##.c`, `world_map.c`)
- **Tilemap dimensions:**
  - Stage room: 20x18 visible tiles (160x144)
  - Scrollable areas: multiples of 32x32 tiles recommended

## 1.4 Animation Data Format
- **Manifest:** `animations.json` per actor family
- Required fields:
  - `name`
  - `frame_indices[]`
  - `fps`
  - `loop` (bool)
  - `interruptible` (bool)
  - `cancel_windows[]` (for combat)
- Build step compiles JSON → `anim_table.c/.h`

Example manifest entry:
```json
{
  "name": "tommy_dash",
  "frame_indices": [40,41,42,43],
  "fps": 18,
  "loop": false,
  "interruptible": false,
  "cancel_windows": [2]
}
```

## 2) Sprite + Animation Requirements (Complete Lists)

## 2.1 Player: Tommy (Required)
Sprite sheet: `tommy_sheet.png` (indexed 4-color)

### Core movement
- `idle_up/down/left/right` (4 anims, 2-3 frames each)
- `walk_up/down/left/right` (4 anims, 4 frames each)
- `run_up/down/left/right` (4 anims, 4 frames each)
- `turn_in_place` (4 anims, 2 frames each)

### Combat
- `bite_1`, `bite_2`, `bite_3` combo chain (3 anims, 3-5 frames each)
- `bite_charge_hold` (1 anim, 2 frames loop)
- `dash_start`, `dash_travel`, `dash_hit`, `dash_recover` (4 anims)

### State/utility
- `hurt_light`, `hurt_heavy`, `knockdown`, `getup`
- `interact`, `push`, `pull`, `carry_idle`, `carry_walk`
- `victory`, `defeat`, `sleep`, `shock_awaken` (intro use)

### Minimum frame count target
- **~140–180 total frames** (after directional reuse optimization)

  - bite hit / bite whiff

  - stagger break
  - menu move/select/cancel
  - puzzle switch on/off
  - lock open

- Per-faction text blip profiles:

## 5) Naming Conventions (Mandatory)

## 5.1 Files
- Sprites: `spr_<group>_<name>_v##.png`
- Tiles: `tile_<biome>_<set>_v##.png`
- Maps: `map_<region>_<stageId>.tmx`
- Audio BGM: `bgm_<context>_<id>.uge`
- Audio SFX: `sfx_<category>_<name>.json` (or engine equivalent)

## 5.2 Animation states
Use snake_case:
- `idle_down`
- `walk_left`
- `bite_2`
- `dash_hit`
- `phase2_transition`

---

## 6) Memory and Performance Budgets (GB-Oriented)

- Per-bank sprite data capped by bank plan (`BANK 1..n`)
- Favor tile reuse and palette swaps over unique large sheets
- Keep active OAM usage under hardware limits per scanline
- Limit simultaneous FX sprites in combat bursts
- Provide LOD-style animation fallback:
  - full (boss scenes)
  - reduced (overworld crowds)

---

## 7) Deliverables Checklist (Definition of Done)

For each asset family, include:
1. Source file (`.aseprite`/`.uge`/`.tmx`)
2. Export file (`.png`/`.json`)
3. Build output (`.c/.h`)
4. Preview sheet (`docs/previews/*.png`)
5. Validation entry in `asset_manifest.csv` with:
   - dimensions
   - frame count
   - palette compliance
   - bank assignment
   - owner/status
